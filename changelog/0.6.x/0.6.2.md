# 0.6.2 — 2026-04-21

Two polish passes: (1) documentation correctness — the prescription that every consumer server adopt the directory-based changelog is now framed as opt-in, and the workflow docs across `CLAUDE.md` / `AGENTS.md` (root + template) and the `polish-docs-meta` / `release` skills are corrected to state that `changelog/unreleased.md` is a pristine format reference (never edited, renamed, or moved); (2) landing page refinement — status strip counts are now anchor links into their sections, connect tabs get a stronger accent gradient + bolder active state, card hover shadow bumped for tactile feedback, minor contrast tuning on subtle text.

Closes [#41](https://github.com/cyanheads/mcp-ts-core/issues/41).

## Changed

- **`polish-docs-meta` skill (v1.5 → v1.6)** — Step 9 reframed with a pattern-choice table (directory-based default vs monolithic opt-out) rather than a hard gate; monolithic `CHANGELOG.md` is now called out as a fine choice for runtime-only consumer servers where the rollup tooling is pure ceremony. The final checklist bullet accepts either pattern. Step 9 also now states explicitly that `changelog/unreleased.md` is a format reference — never edited, never renamed, never moved.
- **`release` skill (v1.4 → v1.5)** — Step 3 retitled "Finalize the Per-Version Changelog File" and rewritten to drop the `git mv changelog/unreleased.md changelog/<series>/<version>.md` workflow. The per-version file is now authored directly at `changelog/<series>/<version>.md`, using `unreleased.md` as a copy source for the layout. Step 3 includes an explicit "do **not** `git mv` or otherwise rename `unreleased.md`" instruction. Checklist bullet updated to match.
- **`CLAUDE.md` + `AGENTS.md` (root and templates)** — Changelog section rewritten. Replaced the "work-in-progress entries live in `changelog/unreleased.md` and are moved into the appropriate series directory on release" misdescription with "at release time, author the per-version file with a concrete version and date, then run `bun run changelog:build` to regenerate the rollup; `changelog/unreleased.md` is a pristine format reference — never edited, never renamed, never moved". Pre-release consolidation paragraph reworded to clarify that betas/RCs live as sub-headers inside the final version's per-version file, not inside `unreleased.md`.
- **`templates/changelog/unreleased.md`** — added a prominent "FORMAT REFERENCE — this file is never edited, never moved, never renamed" banner in both the YAML frontmatter comment and the body `<!-- -->` block, so new consumer servers inherit the read-only framing from day one.
- **Landing page (`src/mcp-server/transports/http/landing-page.ts`)** — status strip count items (Tools / Resources / Prompts) now render as anchor links to their section IDs for one-click nav; `title` attributes added to the auth and protocol-version status items. Connect card tabs get a stronger visual active state: 3px bottom border, bolder font weight on the selected tab, and a subtle accent-softer gradient fill. Card hover shadow bumped from `sm` → `md` for more tactile feedback. Tools section suppresses the sub-heading when only one group is present (avoids redundancy with the section header). Minor contrast tuning on `--fg-subtle` (light + dark) and several subtle-text sites promoted to `--fg-muted` for readability.
