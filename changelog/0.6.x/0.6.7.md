# 0.6.7 — 2026-04-22

Maintenance release. Cleans up a handful of small issues in `templates/` that affected freshly scaffolded servers, patches the `@types/bun` devDep, and corrects a stale `changelog/unreleased.md` reference in the root agent protocol docs that slipped past the 0.6.6 rename.

## Fixed

- **`templates/src/index.ts` now registers the echo MCP App pair** — `echoAppTool` and `echoAppUiResource` were present in `templates/src/mcp-server/` but never wired into `createApp()`, so every freshly-scaffolded server had them as dead code. The init template now ships with both the plain echo definitions and the app-tool + UI-resource pair registered, giving new consumers a working MCP Apps example out of the box (matching the externally-audienced `add-app-tool` skill they already receive).
- **`templates/_.gitignore` — malformed `.vscode` negation rules** — `.vscode/!settings.json` and `.vscode/!extensions.json` were invalid gitignore patterns. `!` only negates at the start of a pattern, so these lines literally tried to ignore files named `!settings.json`. Replaced with the correct `.vscode/*` + `!.vscode/settings.json` / `!.vscode/extensions.json` idiom so freshly scaffolded servers ignore all `.vscode/` content except the two committed workspace files shipped by the template.
- **`templates/CLAUDE.md` / `templates/AGENTS.md` version header mismatch** — both said `**Version:** 0.1.1`, but the template's `package.json` and `server.json` use `0.1.0`. Aligned all four to `0.1.0` so consumers don't see conflicting numbers on day one.
- **Root `CLAUDE.md` / `AGENTS.md` stale `unreleased.md` reference** — the 0.6.6 release renamed `changelog/unreleased.md` to `changelog/template.md` but missed the section-level mention in the Changelog section of the root agent protocol docs. Updated the mention to match the actual file name.
- **`templates/.github/ISSUE_TEMPLATE/bug_report.yml` stale framework-version placeholder** — placeholder showed `0.1.29` from an earlier series. Swapped for a generic `0.x.y` so the placeholder doesn't go stale every release.

## Changed

- **`templates/_.dockerignore` adds `.agents`** — dockerignore already excluded `.claude/` but was missing `.agents/`, inconsistent with `templates/_.gitignore` which excludes both. Keeps agent scratch directories out of the production image regardless of which agent the consumer uses.
- **`templates/package.json` bumps `@types/node` `^25.5.0` → `^25.6.0`** to match the framework's own devDep range.
- **`@types/bun` devDep `^1.3.12` → `^1.3.13`** — routine patch bump from `bun update --latest`.
