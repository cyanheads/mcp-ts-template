# 0.6.1 — 2026-04-21

Refines the landing page shipped in 0.6.0. The hero collapses to an eyebrow + display-size name + single-line status strip, and the connect snippets now live in a tabbed terminal-chrome card (STDIO / HTTP / Claude CLI / curl) with a new `landing.envExample` config for surfacing server-specific env vars in the STDIO JSON block and `--env` flags. Purely additive — no API changes.

## Added

- **`landing.envExample`** (`src/core/serverManifest.ts`) — optional `Record<string, string>` on `LandingConfig` for example env vars shown in the STDIO JSON config and Claude CLI `--env` flags. Values render verbatim in the page HTML — they're placeholders, never real secrets. Max 12 entries, truncated silently beyond that. Normalized into an ordered `Array<{ key, value }>` on the manifest so insertion order is preserved in the rendered block.
- **`landing-env-example-*` linter rules** (`src/linter/rules/landing-rules.ts`) — startup-time validation of `envExample` shape: type check (error), entry count vs `LANDING_MAX_ENV_EXAMPLE` (warn), SCREAMING_SNAKE_CASE key convention (warn), non-string value (error).
- **Tabbed connect card** (`src/mcp-server/transports/http/landing-page.ts`) — terminal-chrome UI (macOS traffic-light dots + endpoint pill) with CSS-only radio-input tabs for STDIO, HTTP, Claude CLI, and curl panels. No JS framework; uses `:has(:checked)` panel switching. The Claude CLI panel expands `envExample` into `--env KEY=value` flags; the STDIO panel renders it as JSON.

## Changed

- **Hero layout** — eyebrow (framework name + protocol version, uppercase mono) + display-size server name (`clamp(2rem, 4.5vw + 0.5rem, 3.5rem)`) + version/pre-release chips + tagline + single-line status strip (auth · tools · resources · prompts · protocol version, separator-dot delimited). Replaces the previous two-column identity + auth banner.
- **Ambient visual layer** — fixed top accent hairline with radial gradient, radial accent glow top-left, and a fine dot grid overlay. All pure CSS, pointer-events: none, zero runtime cost.
- **Color and spacing tokens** — tightened neutral palette (zinc-like scale), added `--bg-code`, `--border-strong`, `--accent-edge`, `--accent-softer`, `--grid-dot`, `--glow-strength`, `--space-20`, `--radius-xs`, `--text-display`, `--duration-slow`. Main column widened to 1120px. Inter + JetBrains Mono preferred when available.
- **Footer** — single-row, dim, separator-dot delimited (was multi-cluster layout with auto-derived GitHub/npm links).
- **Test expectations** — two landing-page DOM assertions updated for the new class structure (`badge-version` moves off `.badge`, `section-count` becomes its own span).
